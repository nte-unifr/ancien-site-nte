Modèles logiques


Afin de clarifier les objectifs et, par là-même occasion, d’évaluer un programme, il est très utile d’établir un modèle logique (« logic model » ou « Théorie de programmes »).
Un modèle logique décrit comment les éléments d’un projet ou d’un programme contribuent à atteindre les objectifs et quelles sont leurs relations réciproques. Ces modèles sont généralement constitués des composants suivants :

  • Ressources (savoir-faire, temps de travail, moyens financiers, etc., tout ce qui entre dans le projet et permet de le réaliser).
  • Mesures (tout ce qui est fait pour réaliser le projet).
  • Produits (ce qui est développé, mis à disposition ou offert, par exemple les technologies, les contenus, les cours etc.).
  • Résultats et effets (ce qui doit être atteint par le programme à court, moyen et long terme, par exemple les compétences en matière de média, la valeur sur le marché du travail, etc.).
  • Des facteurs peuvent en outre être spécifiés (conditions de l'environnement qui peuvent influencer le projet de manière positive ou négative, telles que les contraintes de l’université, les conditions techniques, etc.)

Comment lire un modèle logique ?

Les différents composants sont reliés entre eux par des relations « si–alors » : si certaines ressources sont disponibles, alors les mesures prévues peuvent être mises en œuvre. Si les mesures sont mises en œuvre, alors les produits seront disponibles. Si les produits sont disponibles, alors certains résultats seront atteints, etc.

Vérifiez …

  • si la logique du programme a été clarifiée: qu'est-ce qui doit être atteint avec quels moyens?
  • si les hypothèses sur les relations de cause à effet sont compréhensibles.
  • s’il devient clair comment et pourquoi un programme devrait réussir.
