Analyse du réseau

À l'aide d'une analyse de réseau, on peut examiner les types de relations existant entre les projets, ainsi que la qualité et la densité du réseau de projets au sein d’un programme. On met ainsi en évidence des synergies possibles mais également, le cas échéant, un besoin de mesures de coordination complémentaires (p. ex. des réunions de réseau).

Les différents projets au sein d’un programme représentent les nœuds du réseau, dont on veut montrer les interrelations. Voici des dimensions possibles au sein du réseau :
Orientation : Le projet A connaît le projet B.

Connaissance : Le projet A est connu par le projet B.

Relations d'entrée (Input) : Le projet A peut utiliser un produit du projet B.

Relations de sortie (Output) : Un produit du projet A est pertinent pour le projet B.

Coopération : Le projet A travaille en collaboration avec le projet B.


Une analyse de réseau peut être mise en œuvre de façon simple avec le questionnaire suivant : Fragebogen zur Vernetzung (doc, 20 kB, en allemand). Insérez les projets avec lesquels vous voulez effectuer l'analyse de réseau (nœuds du réseau) dans les lignes. Dans les colonnes, vous retrouvez les dimensions au sein du réseau (à modifier ou compléter si nécessaire).

  • Exemple d'évaluation

    Une possibilité simple consiste à représenter les relations entre les projets relevées grâce au Fragebogen zur Vernetzung en tant que liens entre nœuds (cf. Netzwerkdarstellung 1 ; pdf, 276 kB, en allemand). Une représentation pour chaque dimension : orientation, connaissance etc. 

  • Les dimensions orientation / connaissance et entrées / sorties peuvent en outre être visualisées dans un système de coordonnées : Netzwerkdarstellung 2 (pdf, 26 kB, en allemand). 

  • D'autres possibilités d'évaluation peuvent être obtenues avec des outils d'analyse de réseaux comme p. ex. UCINET (http://www.analytictech.com/ucinet/ucinet.htm).
