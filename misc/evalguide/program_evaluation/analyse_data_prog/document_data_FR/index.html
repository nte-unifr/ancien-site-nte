Saisir les données

Lorsqu’on utilise des données issues des évaluations des projets pour l'évaluation du programme, ces données devraient être documentées par les évaluateurs des projets dans un format approprié et décidé à l'avance. Pour l'évaluation du programme, on peut utiliser soit les données brutes issues des projets (la feuille de données d’origine, voir plus bas), soit des statistiques descriptives (voir page Evaluer les données - statistique descriptive).

Saisie des données d'un questionnaire
Les explications qui suivent concernent les questionnaires composés de questions fermées (choix entre des réponses données). Afin de pouvoir évaluer les données d’un tel questionnaire avec un programme de statistiques, p. ex. SPSS ou Excel, vous devez d'abord créer une fiche de données (cf. Beispiel-Datenblatt, pdf, 36 kB, en allemand). Celle-ci est composée de colonnes et de lignes.

=> Dans les colonnes se trouvent les questions (idéalement dans le même ordre que dans le questionnaire) ; la première colonne devrait être destinée à un numéro d'identification des participants, ou un autre identifiant.
=> Les lignes contiennent les réponses des participants aux différentes questions. 
Participant Question 1 Question 2 Question 3 Question 4 etc.
1
2
3
4

Veillez à ce que vous sachiez à tout moment quel titre de colonne renvoie à quel item dans le questionnaire (c.-à-d. à la question respective). Pour cela, il est possible d'attribuer à chaque question un identifiant (abréviation) et de le noter.

Codification des réponses
Pour insérer les réponses dans la feuille de données et pour les analyser, on code les réponses : à chaque réponse proposée est attribué un nombre (p. ex. 1 pour "n'est pas le cas", 2 pour "n'est plutôt pas le cas", 3 pour "est parfois le cas", 4 pour "est souvent le cas" et 5 pour "est le cas" ; ou 1 pour "masculin" et 2 pour "féminin", ou encore 1 pour "oui" et 0 pour "non" ; etc.) La signification de ces codes devrait, elle aussi, être bien documentée !

IMPORTANT : Plusieurs questions sur le même thème (p. ex. 4 questions sur un aspect de l'utilisabilité, le « pilotage ») peuvent former ce qu’on appelle une échelle. Pour l'analyse, on ne regarde habituellement pas chaque question séparément, mais on calcule une moyenne des questions dans la même échelle. Pour cela, il est important d'inverser le codage pour les questions formulés négativement.

moyenne

« La moyenne est un indice autorisé uniquement pour les caractéristiques quantitatives. A la différence de la => médiane, elle est fortement influencée par les valeurs extrêmes ; il est par conséquent conseillé de l’utiliser uniquement pour des valeurs distribuées de façon symétrique. Elle se rapporte toujours à un => échantillon ; dans le cas d’une => population, on parle de valeur attendue. » (traduit de : Statistik-Lexikon, Mittelwert)

Exemple :

 Item formulé positivement : "Le logiciel permet de suspendre le travail à n'importe quel point et de continuer depuis celui-ci plus tard, sans pertes"
Codes: 1 pour "n'est pas le cas" - 5 pour "est le cas".
Item formulé négativement : "Le logiciel impose un respect inutilement rigide des étapes de travail." Codes : 5 pour "n'est pas le cas" - 1 pour "est le cas".

Valeurs manquantes
A un item non coché (non-réponse), il ne faut surtout pas attribuer le code "0" car lors d'un éventuel calcul de la moyenne, celui-ci fausserait le résultat vers le bas. Laissez simplement la case vide.
