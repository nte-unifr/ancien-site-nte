Logic models


Developing what are known as logic models (also 'program theory') proves to be very useful when it comes to goal clarification and thus for project or program evaluation. A logic model describes how the parts of a project or program contribute to achievement of the goals and how they interrelate. The models usually consist of the following components:

  • Resources (know-how, hours of work, funds etc., everything which goes into the project and makes it possible)
  • Measures (everything done to implement the project)
  • Products (what is developed, made available or offered, e.g. technologies, content, courses, etc.)
  • Results and effects (what the project should achieve in the short, medium or long-term, e.g. media skills, readiness for employment, etc.)
  • Factors can also be specified (environmental conditions which can have a positive or negative effect on the project such as requirements of the university, technical conditions, etc.)

How are logic models read?
The various components are in "if-then" relationships: If certain resources are available, then the planned measures can be implemented. If the measures are implemented, then the products will be available. If the products are available, then certain outcomes are achieved, etc.


Please check whether …

  • the logic of the project is clear: What should be achieved with which means?
  • the assumptions on the cause-effect relationships are clear.
  • it is clear how and why a project or program should succeed.
