Saisir les données

Saisie de données de questionnaires
Les explications qui suivent concernent les questionnaires composés de questions fermées (choix entre des réponses données). Afin de pouvoir évaluer les données d’un tel questionnaire avec un programme de statistiques tel que SPSS ou Excel, vous devez en premier lieu créer une fiche de données (cf. Beispiel-Datenblatt, pdf, 36 kB, en allemand). Celle-ci est composée de colonnes et de lignes.

=> Dans les colonnes se trouvent les questions (idéalement présentées dans le même ordre que dans le questionnaire) ; la première colonne devrait être destinée à un numéro d'identification du participant.
=> Les lignes contenant les réponses des participants aux différentes questions. 
Participant
 Question 1
 Question 2
 Question 3
 Question 4
 etc.
1          
2          
3          
4          

Veillez à ce que vous sachiez à tout moment quel titre de colonne correspond à quel item dans le questionnaire (c.-à-d. la question respective). Pour cela, il est possible d’attribuer un identifiant (abréviation) à chaque question et de le noter dans un document à part.

Codification des réponses
Pour saisir les réponses dans la feuille de données et pour les analyser, on code les réponses respectives, donc à chaque réponse proposée est attribué un nombre (p. ex. 1 pour "n'est pas le cas", 2 pour "n'est plutôt pas le cas", 3 pour "est parfois le cas", 4 pour "est souvent le cas" et 5 pour "est le cas"; ou 1 pour "masculin" et 2 pour "féminin", ou encore 1 pour "oui" et 0 pour "non"; etc.) La signification de ces codes devrait, elle aussi, être bien documentée !

IMPORTANT : Plusieurs question sur le même thème (p. ex. 4 question sur un aspect de l'utilisabilité, la « possibilité de pilotage »,) forment ce qu’on appelle une échelle. Pour l'analyse, on ne regarde habituellement pas chaque question séparément, mais on calcule une moyenne des questions qui sont dans la même échelle. Pour cela, il est important d'inverser le codage pour les questions formulées négativement. Exemple :

moyenne

« La moyenne est un indice autorisé uniquement pour les caractéristiques quantitatives. A la différence de la => médiane, elle est fortement influencée par les valeurs extrêmes ; il est par conséquent conseillé de l’utiliser uniquement pour des valeurs distribuées de façon symétrique. Elle se rapporte toujours à un => échantillon ; dans le cas d’une => population, on parle de valeur attendue. » (traduit de : Statistik-Lexikon, Mittelwert)

 

Item formulé positivement : "Le logiciel permet d'interrompre le travail à n'importe quel point et de continuer depuis celui-ci plus tard sans pertes" Codes : 1 pour "n'est pas le cas" - 5 pour "est le cas".
Item formulé négativement: "Le logiciel impose un respect inutilement rigide des étapes de travail." Codes : 5 pour "n'est pas le cas" - 1 pour "est le cas".

Valeurs manquantes
A un item non codé (non-réponse), il ne faut surtout pas attribuer le code "0", car lors d'un éventuel calcul de la moyenne, celui-ci fausserait le résultat vers le bas. Laissez simplement la case vide.
